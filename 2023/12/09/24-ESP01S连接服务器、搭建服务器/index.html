ESP01S连接服务器、创建服务器

本例程中使用的ESP01S的固件为默认固件

1、ESP01S连接电脑服务器

首先需要确保ESP01S和PC处于同一网络下（连接同一个WiFi），才能进行连接！因为本例程中连接的IP地址是私网IP地址，如果使用公网IP地址只需要ESP01S连接到可以上网的热点即可。

1.1、相关AT指令

AT指令 返回 说明
AT OK AT测试指令，正常回复”OK”
AT+CWMODE=1 OK 设置esp8266为客户端模式（station)
AT+RST OK 复位保存配置
AT+CWJAP=“WiFi名称” ,“WiFi密码” WIFI CONNECTED WIFI GOT IP 配置ESP01连接相应的WiFi
AT+CIPMUX=1 OK 配置为多链接模式
AT+CIPSTART=0,“TCP”,“你的服务端IP”,PORT 0,CONNECT OK 进行TCP连接，0为当前客户端号码，”TCP”为通讯协议,服务端IP在本项目中为电脑的IP，PORT为端口号，本项目中为8888
AT+CIPSEND=客户端号码,length OK 向服务端发送length长度的数据

1.2、具体操作

电脑监听端口

使用网路调试助手进行服务器端口监听，具体配置如下：本地主机地址请查看

本地主机地址如果有多个，可以通过设置->WLAN->硬件属性中查看（图中下面的ip地址跟上面不同无需理会，截图时改了连接的WiFi因此ip不同，实际操作中要相同）

ESP01S连接服务器

打开串口调试助手，选择相应的串口号，波特率根据实际选择，一般为115200，打开串口，按顺序进行AT指令配置连接

  1. 先设置为Station客户端模式
  2. 连接跟电脑一样的WiFi上
  3. 打开多链接
  4. 进行TCP连接

如果没有打开多链接的话，使用AT+CIPSTART=0,"TCP","192.168.85.231",8888可能会出错，单链接的话应该使用AT+CIPSTART="TCP","192.168.85.231",8888，即去掉客户端号码即可，在进行链接（特别提醒AT指令中的双引号逗号都是英文格式的，不要打错）

通过以上操作即可成功与电脑的8888端口进行TCP通讯！

2、电脑连接ESP01S服务器

电脑需要连接到ESP01S发出的WiFi热点

2.1、相关AT指令

AT指令 返回 说明
AT OK AT测试指令，正常回复”OK”
AT+CWMODE=2 OK 设置esp8266为服务端模式（AP)
AT+RST OK 复位保存配置
AT+CWSAP=”要设置的WiFi名称”,”WiFi密码”,通道号,保密方式 OK 配置ESP01S本身发出的WiFi名称和密码，通道号范围0-14,加密方式是4即WPA_WPA2_PSK（此时需要主要WiFi密码至少要8位才能设置成功），也可为0直连
AT+CIPMUX=1 OK 配置为多链接模式
AT+CIPSERVER=1,8888 OK 进行TCP连接，0为当前客户端号码，”TCP”为通讯协议,服务端IP在本项目中为电脑的IP，PORT为端口号，本项目中为8888
AT+CIFSR IP地址信息 查询ESP01S的IP地址

2.2、具体操作

ESP01S创建端口监听

通过AT指令配置ESP01S为服务端模式，并监听8888端口

电脑连接服务器

依然使用网络调试助手，在电脑已经连接到ESP01S的AP热点的情况下，选择TCP Client客户端协议，远程主机地址即为上面通过AT指令AT+CIFSR看到的IPv4地址，端口依然为8888，然后打开连接即可

通过上面的步骤，就可以实现ESP01S与电脑之间的相互通讯了，两者互为服务端客户端都行

数据传输

数据传输主要通过以下两个AT指令

AT指令 返回 说明
AT+CIPMODE=1 OK 设置ESP01S为透传模式
AT+CIPSEND > 开启数据传输

透传模式即为无需提取声明数据长度，无需提取通过AT+CIPSEND=?指定数据长度，然后在>后输入相应长度的数据，开启透传模式后，只需要在通过AT+CIPSEND开启数据传输，在回复>后即可直接发送任意数据，无需在下次发送前重新发送AT+CIPSEND=?指定长度在发送数据，大白话就是透传模式下，发送开启数据传输后，就可以跟微信聊天一样，你给ESP01S发什么，ESP01S就把数据直接发给服务器，并且如果有数据进来也不会显示成”+IPD,0,,”的格式，而是会直接把数据原封不动在从ESP01S串口输出

可以通过AT+CIPMODE?查询当前是否处于透传模式
